The goal of React Fiber is to increase its suitability for areas like animation, layout, and gestures. Its headline feature is incremental rendering: the ability to split rendering work into chunks and spread it out over multiple frames.


Other key features include the ability to pause, abort, or reuse work as new updates come in; the ability to assign priority to different types of updates; and new concurrency primitives.

‚≠ê 1. Virtual DOM

Virtual DOM = Browser DOM ka lightweight copy (memory me).

React real DOM ko directly modify nahi karta (slow hota hai).

Pehle virtual DOM banata hai ‚Üí phir compare karke sirf jaha change hua ho wahi real DOM update karta hai.

Isse app fast chalti hai.

üëâ Example:
Tumhare room ka photo (virtual DOM) le kar changes mark karna ‚Üí phir room me sirf wahi changes apply karna ‚Üí pura room rebuild nahi karna.

‚≠ê 2. Reconciliation

Reconciliation = React ka process jisme wo naya virtual DOM aur purana virtual DOM compare karta hai.

Matlab:

Pehle app ka ek virtual DOM hota hai

Update hone par naya virtual DOM banta hai

React dono compare karta hai (diff algorithm)

Sirf changed parts ko update karta hai

üëâ List me key isliye important hoti hai taaki React ko pata chal sake kaun sa item same hai aur kaunsa new.

‚≠ê 3. React Fiber

Fiber = React ka naya engine jo updates ko chote-chote parts me tod kar handle karta hai.

Old React:

Ek update aaya ‚Üí pura kaam ek baar me karta tha ‚Üí UI freeze ho sakti thi

React Fiber:

Kaam ko small pieces (fibers) me todta hai

Kabhi bhi pause kar sakta hai

Kabhi bhi resume kar sakta hai

High priority tasks (animations, user typing) ko pehle karta hai

Low priority work baad me karta hai

üëâ This makes React smoother, faster, aur better scheduling wala.
