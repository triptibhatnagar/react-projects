### USESTATE, USEEFFECT, USECALLBACK - HOOKS USED IN THIS PROJECT

1. useState
- Purpose:
    Component ke andar state (data) ko store aur update karne ke liye.
- Important points:
    useState() ek value aur usko update karne ka function deta hai.
    State change hoti hai → component re-render hota hai.
- Syntax:
    const [value, setValue] = useState(initialValue);
- Har state independent hoti hai (multiple useState allowed).

2. useEffect
- Purpose:
    Component ke render hone ke baad side effects chalane ke liye.
    (Side effects = API call, timer, event listener, subscriptions)
- Important points:
    Runs after render.
    Dependency array decide karta hai kab chalega:
    - useEffect(() => {}) → har render ke baad
    - useEffect(() => {}, []) → sirf 1 baar
    - useEffect(() => {}, [state]) → jab state change ho
    - Cleanup function (return () => {}) un effects ko stop/cleanup karta hai.

- Syntax:
    useEffect(() => {
    // effect
    return () => {
        // cleanup
    };
    }, [dependencies]);

3. useCallback
- Purpose: 
    Ek function ko memoize (yaad) karna taaki wo har render me dobara na bane.
- Important points:
    Performance optimization ke liye use hota hai.
    Especially useful jab function child component ko pass kar rahe ho.
    Agar dependencies nahi badalti, function same rehta hai.
- Syntax:
    const fn = useCallback(() => {
    // function code
    }, [dependencies]);

- dependencies:
    ki inn inn se baat ho rahi hai
